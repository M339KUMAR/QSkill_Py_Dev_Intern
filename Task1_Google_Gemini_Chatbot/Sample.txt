Suppose the conversation is:
#-----------------------------
User:
Hello

Assistant:
Hi!

User:
My name is Praveen.
#-----------------------------------
The history sent to Gemini becomes:
#-----------------------------------
[
    {
        "role": "user",
        "parts": [
            {
                "text": "Hello"
            }
        ]
    },

    {
        "role": "assistant",
        "parts": [
            {
                "text": "Hi!"
            }
        ]
    },

    {
        "role": "user",
        "parts": [
            {
                "text": "My name is Praveen."
            }
        ]
    }
]

Gemini now receives the entire conversation and can answer context-aware questions.

Before Lesson 5: 

The memory implementation above is a great learning exercise, but the latest Google Gemini SDK also provides a Chat Session API that automatically manages conversation history. It's cleaner, more efficient, and closer to how production applications are built.

Since your goal is to build an industry-level project, I recommend that in Lesson 5 we refactor our chatbot to use the SDK's chat session while also enabling Google Search grounding for real-time information. 

This will give you cleaner code and satisfy the remaining internship requirement in a way that's aligned with current best practices.
